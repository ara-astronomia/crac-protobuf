# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Install dependencies
uv sync

# Regenerate Python gRPC bindings from .proto files
uv run python generate_proto_code.py

# Build the distributable package
uv build
```

There are no tests or linting configured in this project.

## Architecture

This is a Python library that ships generated gRPC/protobuf bindings for astronomical observatory hardware control (telescopes, cameras, roofs, curtains, UPS, weather, etc.).

**Source of truth:** `interfaces/*.proto` — the only files to edit by hand.

**Generated output:** `crac_protobuf/` — every `*_pb2.py` and `*_pb2_grpc.py` is regenerated by `generate_proto_code.py` (uses `sys.executable` for environment independence) and must never be edited directly.

### Proto dependency graph

`camera.proto`, `telescope.proto`, `curtains.proto`, and `roof.proto` all import `button.proto` for shared UI element types. `ups.proto` imports `chart.proto`.

### Workflow for proto changes

1. Edit or create a `.proto` file in `interfaces/`
2. Run `uv run python generate_proto_code.py` to regenerate `crac_protobuf/`
3. Bump the version in `pyproject.toml`
4. Commit both the `.proto` source and the regenerated `_pb2*.py` files

### Backward compatibility rules

- **Never delete or renumber fields** — use `reserved` if a field is no longer needed.
- GUI responses: standardize on `repeated ButtonGui buttons = N;` (not singular `button_gui`).
- `SetAction` RPC is for operational commands only; reads should use a dedicated `GetStatus` method.

### Release

Publishing to PyPI is automated via `.github/workflows/python-publish.yml` and triggers on GitHub release creation.
